## Codebase Patterns
- youmu Rust extension is now available (ParquetReaderCachePy imports successfully).
- youmu submodule commits must happen inside `submodules/youmu/` first, then `git add submodules/youmu` in parent repo.
- LIBERO 64KB dataset at `/mnt/local/localcache00/libero_64KB/` uses episode-per-file layout: `data/chunk-NNN/episode_NNNNNN.parquet` with flat column names (`state`, `actions`, `image`).
- Episode metadata is at `meta/episodes.jsonl` (JSONL format), NOT `meta/episodes/chunk-000/file-000.parquet`.
- Column names in actual dataset: `state` (not `observation.state`), `actions` (not `action`), `image` (not `observation.images.image`).
- LiberoYoumuDataset defaults now match actual LIBERO schema: `state_column='state'`, `action_column='actions'`, `image_column='image'`.
- For episode-per-file layout, `_load_chunk` filters JSONL metadata by parsing episode indices from filenames in each chunk directory.
- `load_libero_task_descriptions` auto-detects format by file extension: `.jsonl` → JSONL loader, otherwise → parquet loader.
- Training script metadata resolution order: `meta/episodes.jsonl` → `meta/episodes/episodes.parquet` → `meta/episodes/chunk-000/file-000.parquet`.
- Both `/mnt/local/localcache00/libero` and `libero_64KB` have identical data; 64KB is a page-size re-encoding only.
- `huggingface_hub`, `av`, and `lerobot` are now installed and importable.
- The youmu __init__.py imports from `.youmu` (Rust extension). Since the Rust extension is available, import `LiberoYoumuDataset` and `_find_physical_column_index` directly — no mock patching needed.
- The 64KB dataset still has dict-encoded columns (RLE_DICTIONARY), causing the Rust reader to panic. LiberoYoumuDataset auto-falls-back to PyArrow in this case.
- Synthetic parquet PNG images must be valid — use Pillow-generated PNGs, not hand-crafted bytes. The old minimal PNG bytes were corrupt.
- Importing `veomni.data.multimodal.data_transform` triggers the full veomni import chain (torch, transformers, huggingface_hub). Use `importlib.util.spec_from_file_location` with mocked dependencies for isolated tests.
- Synthetic test parquet files can be created with PyArrow `FixedSizeListArray.from_arrays` and `StructArray.from_arrays`.
- Use plain/non-dict-encoded columns in synthetic parquets to avoid Rust reader panics.
- ruff-format pre-commit hook reformats files automatically; always re-stage after first commit attempt fails due to formatting.
- Synthetic parquet files must use `write_page_index=True` for the youmu Rust reader's offset index requirement.
- lerobot 0.4.4 uses codebase v3.0 format; on-disk dataset is v2.0. Must create temp v3.0 wrapper directory (symlink data, convert JSONL metadata to parquet) for lerobot loading.
- DO NOT import `tasks/omni/train_qwen_vl_libero.py` via importlib — it triggers full veomni import chain which fails due to transformers/huggingface-hub version mismatch. Build datasets directly.
- youmu key names: `observation.state`, `action`, `observation.images.image`; lerobot key names: `state`, `actions`, `image`.
- youmu action semantics: returns actions for frames [t+1, ..., t+pred_len]; lerobot delta_timestamps must start at 1/fps to match.
- youmu `frame_index` is a global cumulative index across all episodes, maps directly to lerobot's sequential dataset index.
- youmu excludes last pred_len frames per episode (anchor frames only); lerobot includes all frames with padding. Length difference = pred_len × num_episodes.
- youmu images: uint8 (obs_len, H, W, C); lerobot images: float32 (obs_len, C, H, W). Convert via `(img.permute(0,2,3,1) * 255).to(uint8)` for comparison.
